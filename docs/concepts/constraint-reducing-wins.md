# Constraint-Reducing Wins — Daily Operating Pattern

## Definition
A quick, measurable, low-risk improvement that removes a bottleneck or accelerates progress on a KPI.

**Scope:** ~90 minutes or less  
**Frequency:** Daily (part of Heartbeat ritual)  
**Goal:** Compound momentum through systematic constraint removal

## Anatomy of a Win

### 1. Identify the Constraint
Look at today's #1 bottleneck:
- What's blocking progress on the scorecard?
- What would unblock the most important thing?
- What could be solved in <90 min?

### 2. Solve It (Stay in Lane)
- If approved lane (e.g., Content, Outreach): use pre-approved templates + constraints
- If new territory: ask for constraint boundaries first
- If reversible + low-risk: execute and report

### 3. Report the Win
**Action Log format:**
- **What I did / changed** — Specific action, file path, measurable change
- **Where** — File path, tool, system affected
- **Why** — How this improves the KPI / removes bottleneck
- **Rollback** — How to undo if needed
- **Next** — What's the next bottleneck?

## Metrics to Track

For each week's wins, note:
- **Count:** How many constraints reduced?
- **KPI moved:** Which scoreboard metric improved?
- **Time invested:** Total hours spent
- **Next bottleneck:** What emerges after this win?

## Why This Works

1. **Momentum compounds** — 7 wins/week × 4 weeks = 28 removals/month
2. **Reduces big meetings** — Solve before asking; decision friction disappears
3. **Keeps focus narrow** — One constraint per day prevents thrash
4. **Builds autonomy** — Renzo sees execution capability, trusts delegation
5. **Low risk** — Always reversible; easy to rollback if wrong

## Anti-Patterns (Don't Do This)

- ❌ "Work on a big project all day" (no daily win reported)
- ❌ "Fix minor typo and call it done" (not materially useful)
- ❌ "Ask permission to ship a win" (violates autonomy model)
- ❌ "Skip the Action Log" (invisible execution, no compounding)
- ❌ "Ship work that creates debt for Renzo" (cleanup is the opposite of help)
