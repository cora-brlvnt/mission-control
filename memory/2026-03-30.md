### 2. Escalation Loop Anti-Pattern (Prevention Rules)
- **Pattern:** 20 days of increasingly urgent messages → zero engagement = waste
- **Root cause:** Decision-making friction (not communication gap)
- **Prevention:** Single deadline, honor it, redirect energy
- **Learning:** Silence = data; escalating into silence = waste

### 3. Async Systems Debt Accumulation
- **Problem:** Dead code accumulates silently (no failure signal)
- **Case study:** 3.6M tokens/day waste from Feb 22–Mar 29 (redundant cron)
- **Detection requires:** Quarterly audits (not error-based)
- **Prevention:** Document purpose of every cron job + quarterly enumeration

### 4. Autonomous Operating Model Requirements
- Pre-defined boundaries (no ambiguity)
- Binary decision frames (no open-ended questions)
- Hard SLAs with automatic consequences
- Quarterly infrastructure audits
- Action logging for every decision

### 5. Window-Based Planning Non-Negotiability
- Sprint windows tied to infrastructure constraints
- Missing gate cascades into full cycle deferral (March 27 → April 1 re-plan)
- Mechanical (not motivational): Calendar is the constraint, not urgency
