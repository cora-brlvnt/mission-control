## Comparative Analysis: Why Soft SLAs Fail, Why Hard SLAs Work

### Soft SLA Pattern (March 17 baseline)

**Characteristics:**
- Vague deadline ("decide this week")
- No explicit consequence ("impacts downstream")
- Long window (10+ days permits drift)
- Escalation-dependent (success depends on persuasion)

**Failure mechanics:**
1. Deadline passes → no escalation trigger (soft = optional)
2. Escalation messages sent → silence treated as "not now, later" (not "no")
3. Repeated escalation into silence → wastes communication capital
4. System converges on indefinite deferral (soft deadline loop never closes)

**Outcome:** March 17 deadline → 25+ days overdue (no mechanical consequence)

### Hard SLA Pattern (April 1 baseline)

**Characteristics:**
- Explicit deadline (April 3, 12:04 AM EDT)
- Quantified consequence ($300K+ at-risk value)
- Short window (48 hours, creates mechanical urgency)
- Automatic escalation (consequence triggers regardless of persuasion)

**Success mechanics:**
1. Deadline approaches → mechanical urgency builds (not discretionary)
2. Binary frame forces explicit choice (Approve / Revise / Defer)
3. Automatic consequence if missed (April 8 re-plan activates non-negotiably)
4. System converges on decision (hard deadline loop must close)

**Expected outcome:** April 1 hard SLA + April 3 deadline → Decision expected within 48–168 hours

## Why Escalation Messages Into Silence Are Waste

### The Pattern

```
Message 1: "Decision needed"              → Silence
Message 2: "Decision still needed"        → Silence
Message 3: "Escalating, decision urgent"  → Silence
Message 4: "Decision critical, impacts X" → Silence
Message 5–8: Increasing urgency          → Silence
                                          ↓
Result: Zero engagement, $2,000+ sunk cost, wasted relationship capital
```

### Why It Fails

1. **Silence is data, not delay** — After message 1 with zero response, continued messages signal "not now, later" or "delegated to someone else," not "busy, will respond."

2. **Escalation fatigue sets in** — 8 messages over 8 days creates perception of spam, not urgency. Inverse relationship: more messages = less saliency per message.

3. **No forced choice** — Escalation messages ask for response, but don't force explicit decision. Recipient can stay silent indefinitely.

4. **Relationship capital depletes** — Each message signals "you're not responding," eroding goodwill. By message 5, tone becomes accusatory (even if written neutrally).

### Better Approach (Hard SLA)

```
Message 1: "Decision required by April 3, 12:04 AM EDT"
           "Options: Approve / Revise / Defer"
           "If no decision by April 3, April 8 re-plan activates automatically"
           
April 2:   (monitoring, no follow-up message)

April 3:   (gate closes, decision logic executes)
           "No decision received by April 3 deadline"
           "April 8 emergency re-plan cycle activated"
           
Result: Mechanical consequence, zero negotiation, relationship capital preserved
```

**Why it works:**
- One deadline per cycle (not repeated escalation)
- Binary frame (not open-ended ask)
- Automatic consequence (not persuasion-dependent)
- Gate closes on specified date (not indefinite waiting)
